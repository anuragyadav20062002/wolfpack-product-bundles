# Business Requirement: Promo Banner Image Crop Tool

**Feature ID:** promo-banner-crop
**Created:** 2026-02-20
**Parent Feature:** bundle-images-gifs

---

## Executive Summary

Merchants who upload wide-format images for the promo banner often have images whose focal point (logo, product, text) is not centered. This feature adds an "Adjust Image" crop selector directly on the configure page so merchants can choose exactly which region of the image is displayed in the banner, without needing external image editing tools.

---

## Problem Statement

**Who:** Merchants who have uploaded a promo banner background image via the Images & GIFs section.

**Pain point:** The promo banner CSS uses `background-size: cover; background-position: center`. If the merchant's image focal point is off-center (e.g., a logo in the top-left, a product shot on the right), the displayed region in the storefront banner will clip it. There is currently no way to adjust the visible region without editing the image externally, re-uploading, and hoping the result looks right.

**Expected experience:** After selecting a background image, a merchant should be able to open a simple crop editor directly in the configure page, drag a fixed-aspect bounding box over the image to choose the visible region, and save. The storefront widget then applies that crop region as CSS `background-position` / `background-size` values.

---

## Research Findings

### Approach A: CSS-only offset storage (recommended) ✅

Store the crop as `{ x, y, width }` in percentage values. The widget maps these to `background-position` and `background-size` CSS properties at runtime — no server-side transformation needed.

**Pros:**
- No additional storage cost (< 50 bytes JSON)
- Works with any image URL including Shopify CDN URLs (no CORS pixel access required — CSS only)
- Deterministic and reversible — removing crop just deletes the JSON field
- Zero dependency on external libraries

**Cons:**
- Precision limited by CSS percentage resolution (acceptable for banner backgrounds)

**Verdict: ✅ Recommended**

### Approach B: Server-side crop + new CDN URL

Send crop coordinates to the server, use sharp/jimp to crop the image server-side, re-upload the cropped version to Shopify Files.

**Pros:**
- Exact crop result
- Simpler CSS (no coordinate math)

**Cons:**
- Requires CORS-accessible image URL or proxy re-download
- Adds processing time and complexity
- Shopify CDN URLs may block server-side fetch without signed tokens
- Creates a new file per crop edit (storage accumulates)

**Verdict: ❌ Rejected** — too complex for a banner background where CSS precision is sufficient

### Approach C: External crop library (e.g., react-image-crop, cropperjs)

Drop in a ready-made crop UI library.

**Pros:**
- Polished UX out of the box
- Handles touch, resize handles, keyboard

**Cons:**
- Bundle size increase (~50-150 KB gzip)
- External dependencies complicate upgrade paths
- Overkill for a single fixed-aspect-ratio use case
- Still requires Approach A for storage and widget application

**Verdict: ❌ Rejected** — custom implementation covers the requirement with zero bundle overhead

## Functional Requirements

- **FR-01:** After selecting a promo banner image, an "Adjust Image" button appears in the FilePicker selected-state UI
- **FR-02:** Clicking "Adjust Image" opens a full-screen-overlaid crop modal showing the full image
- **FR-03:** The modal displays a draggable bounding box with a fixed 16:3 aspect ratio
- **FR-04:** The area outside the bounding box is dimmed with a semi-transparent overlay
- **FR-05:** Dragging the bounding box repositions it over the image; it stays within image bounds
- **FR-06:** Clicking "Confirm" saves the crop region as `{ x, y, width }` percentage values
- **FR-07:** Clicking "Cancel" or pressing Escape closes the modal without saving
- **FR-08:** The configure page save flow persists `promoBannerBgImageCrop` to the DB alongside `promoBannerBgImage`
- **FR-09:** The widget reads the crop data and applies computed `background-position` and `background-size` values inline on the `.promo-banner` element
- **FR-10:** When `promoBannerBgImageCrop` is null/absent, widget falls back to `background-size: cover; background-position: center`

## Non-Functional Requirements

- **NFR-01:** No external npm dependencies added for the crop UI
- **NFR-02:** The crop editor must work with Shopify CDN image URLs (CORS-safe: CSS rendering only, no canvas pixel manipulation)
- **NFR-03:** Existing bundles without crop data are unaffected — default CSS cover/center applies
- **NFR-04:** Crop modal must render above the App Bridge configure page modal (z-index > 99999)
- **NFR-05:** The crop JSON field adds < 50 bytes to the `bundle_ui_config` metafield payload

---

## User Stories (high level)

- As a merchant, I want to drag a crop window over my banner image so that the most relevant part of the image is shown in the promo banner
- As a merchant, I want to be able to reset or ignore the crop so the banner falls back to a centered, covered background
- As a shopper, I want the promo banner to display the image region the merchant intended, regardless of the source image dimensions

---

## Scope

### In Scope
- "Adjust Image" button in FilePicker selected state
- Crop modal with fixed-16:3-ratio draggable bounding box
- `Bundle.promoBannerBgImageCrop String?` Prisma field
- Crop stored in `bundle_ui_config` metafield
- Widget applies CSS background-position/background-size from crop data
- "Reset crop" implicit (clearing crop data falls back to cover/center)

### Out of Scope
- Server-side image cropping
- Free-aspect-ratio crop
- Crop for images outside the promo banner
- Resize handles on the bounding box (drag-only, fixed size)
- Pinch-to-zoom on the image in the modal
- Undo/redo history

---

## Risks

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Shopify CDN images load slowly in modal (large originals) | Medium | Show a loading spinner while the image loads; size with `width` URL param |
| Touch drag UX on mobile is poor | Low | The configure page is merchant-facing (desktop-first) |
| CSS background-position math edge cases at extreme crop positions | Low | Clamp x/y to valid CSS percentage range (0–100) |
