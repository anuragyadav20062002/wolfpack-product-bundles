# [BR] Business Requirement: Multilingual Discount Messaging

**Document Type:** Business Requirement (BR)
**Feature:** Multilingual Discount Message Configuration
**Date:** 2026-02-19
**Status:** Draft
**Priority:** 🟡 Medium
**Prepared by:** Business Analyst

---

## 1. Business Context

Wolfpack Product Bundles serves Shopify merchants whose stores operate in multiple languages and sell to international customers. A merchant in France, Germany, Japan, or Brazil may configure discount bundles — but the discount messages displayed to their buyers (`"Add 2 more items to get 20% off"`) currently appear only in English, regardless of what language the buyer's browser and Shopify storefront are set to.

This creates a poor buyer experience and reduces conversion on discounted bundles for non-English stores.

---

## 2. Problem Statement

Discount messaging is the most buyer-visible text in the Wolfpack bundle widget. Currently:

- All discount messages are configured as a single English template per rule.
- The storefront widget renders the same English text regardless of the buyer's locale.
- Merchants who run multilingual stores have no mechanism to configure messages in their buyers' languages.
- Competitors (bundle apps) support multilingual messaging, creating a feature gap.

---

## 3. Business Objective

Enable merchants to **configure discount messaging templates in multiple languages** so that buyers see discount messages in their native language, increasing comprehension, trust, and conversion rates on bundled products.

---

## 4. Scope

### In Scope
- A **language selector dropdown** in the Discount Messaging subsection of the Discount & Pricing section on the bundle configuration page.
- The dropdown lists languages by their **IETF shorthand language code** (e.g., `en`, `fr`, `de`, `es`, `ja`).
- Merchants can configure the **"Discount Text"** and **"Success Message"** templates independently for each language.
- The storefront widget **automatically serves the correct language** message based on the buyer's locale (`window.Shopify.locale`).
- A **fallback to English** (`en`) when no template is configured for the buyer's locale.
- All existing bundle configurations remain valid — English messages are preserved without data migration.

### Out of Scope
- Automatic machine translation of messages.
- Translation of the admin app UI itself (DCP labels, navigation, etc.).
- Checkout UI extension messaging (that shows pricing breakdown, not configurable templates).
- Full-page bundle widget (separate implementation if needed later).
- RTL language layout adjustments.
- Language-specific formatting of template variables (currency/number formatting already handled by the widget's variable substitution engine).

---

## 5. User Stories

### Primary User Story
> **As a merchant** running a multilingual Shopify store,
> **I want to** configure my bundle discount messages in multiple languages,
> **So that** my buyers see relevant, locally-written discount messaging in their own language, improving their shopping experience and my conversion rate.

### Supporting Stories

**US-01:** As a merchant, I want to select a language from a dropdown in the Discount Messaging section, so I can configure messages for that specific language.

**US-02:** As a merchant, I want to see the language options listed as standard shorthand codes (e.g., `en`, `fr`, `de`), so I can quickly identify the language I'm configuring.

**US-03:** As a merchant, I want my existing English messages to continue working after this update, so I don't lose any previously configured content.

**US-04:** As a buyer** browsing in French on a multilingual Shopify store, I want to see bundle discount messages in French if the merchant has configured them, so I understand the promotion clearly.

**US-05:** As a buyer whose language hasn't been configured by the merchant, I want to see the English fallback message, so I always see some meaningful messaging rather than a blank.

---

## 6. Functional Requirements

| ID | Requirement |
|---|---|
| FR-01 | The Discount Messaging section SHALL include a language selector dropdown. |
| FR-02 | The dropdown SHALL list available languages by their IETF language shorthand codes. |
| FR-03 | The language list SHALL include at minimum: `en`, `fr`, `de`, `es`, `it`, `ja`, `ko`, `pt`, `zh`, `nl`, `da`, `sv`, `pl`, `cs`, `fi`, `tr`, `nb`. |
| FR-04 | Selecting a language from the dropdown SHALL display the "Discount Text" and "Success Message" fields pre-filled with any previously saved values for that language. |
| FR-05 | The system SHALL save message templates per language, per pricing rule. |
| FR-06 | The storefront widget SHALL detect the buyer's active locale from `window.Shopify.locale`. |
| FR-07 | The widget SHALL serve the message template matching the buyer's language subtag (e.g., `fr` from `fr-FR`). |
| FR-08 | The widget SHALL fall back to `en` if no template is configured for the buyer's language. |
| FR-09 | Existing single-language (English) message configurations SHALL be treated as the `en` language entry with no data loss. |
| FR-10 | The "Show discount messaging" toggle SHALL apply globally, not per-language. |

---

## 7. Non-Functional Requirements

| ID | Requirement |
|---|---|
| NFR-01 | Language switching in the admin UI SHALL be instantaneous (no page reload, no API call on dropdown change). |
| NFR-02 | Saving multilingual messages SHALL reuse the existing save flow (no additional API endpoints required). |
| NFR-03 | Widget locale detection SHALL add < 1ms overhead to bundle render time. |
| NFR-04 | The solution SHALL NOT require any new npm packages for the core feature. |
| NFR-05 | Backward compatibility SHALL be maintained — existing bundle data must not require migration. |

---

## 8. Success Metrics

- Merchant adoption: % of merchants using the feature who configure > 1 language within 30 days.
- Support ticket reduction: Decrease in support requests about non-English messaging.
- Indirect: Conversion rate on bundles in non-English storefronts (tracked via merchant analytics).

---

## 9. Assumptions

- Shopify always sets `window.Shopify.locale` on storefront pages; this is a reliable locale signal.
- Merchants are responsible for translating their own message templates — no machine translation provided.
- The initial language list covers the 17 languages most commonly used in Shopify stores.
- Existing message data format can be extended in place (backward-compatible JSON evolution).

---

## 10. Dependencies

- Existing `BundlePricing.messages` JSON field in the database.
- Existing widget `updateMessagesFromBundle()` function.
- Existing `discountData` save flow in the bundle configuration route.

---

## 11. Risks

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Merchants leaving non-EN languages empty | High | Low | Clear placeholder text, English fallback |
| `window.Shopify.locale` unavailable on some themes | Low | Medium | Fallback chain: Shopify locale → `en` |
| DB JSON size growth for many-language configs | Low | Low | JSON storage is elastic; typical config is < 5 KB |
