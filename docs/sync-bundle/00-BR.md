# Business Requirement: Sync Bundle (Hard Reset)

## Executive Summary

Merchants and developers need a single-click mechanism to fully tear down and re-create all Shopify-side data for a bundle — product, page, metafields, and DB records — so that breaking code changes (schema evolutions, metafield restructures) can be applied to existing bundles without manual Shopify Admin intervention.

## Problem Statement

**Who experiences it:** Developers deploying breaking changes, and merchants whose bundles become broken after app updates.

**Pain points:**

1. **Breaking changes during development** — When the app evolves (e.g., metafield schema changes, cart transform logic changes, Shopify API version bumps), previously created bundles carry stale Shopify-side data. The old `bundle_ui_config`, `component_parents`, `price_adjustment` metafields may have wrong structure. There is currently no way to atomically re-create everything for a bundle.

2. **Out-of-sync state** — A merchant's Shopify product or page can become orphaned (deleted manually on Shopify, or the metafields get corrupted). The app UI shows the bundle as configured, but the storefront widget fails to load.

3. **No recovery path** — The existing "Save" button only updates the DB and pushes metafields. It cannot recover from a missing Shopify product/page. There is no "start over" option for a single bundle without deleting it entirely and recreating from scratch.

## Research Findings

### Approach A: Light Sync (metafield-only re-push)
Re-run the three metafield update operations (`updateBundleProductMetafields`, `updateComponentProductMetafields`, `updateProductStandardMetafields`) from the current DB state. No DB changes, no Shopify object deletion.
- **Pros:** Fast (~2-5s), zero data loss risk, safe to run any time.
- **Cons:** Cannot recover a missing Shopify product/page; does not fix structural corruption in Shopify objects.

### Approach B: Hard Reset (delete + re-create everything)
Delete all Shopify-side objects (product or page, all metafields) and all DB records for the bundle (steps, pricing), then re-create from the existing bundle definition. The bundle's primary ID is preserved so URLs and any external references remain valid.
- **Pros:** Definitively fixes all stale/corrupt state. Handles breaking metafield schema changes. Recovers orphaned Shopify objects.
- **Cons:** Destructive — data loss risk for analytics. Takes longer (~10-20s). Shopify product/page gets a new GID after re-creation.

### Approach C: Two-button UI (Light + Hard)
Expose both modes as separate buttons: "Sync Metafields" (light) and "Hard Reset" (nuclear).
- **Pros:** Gives developers full control.
- **Cons:** UI complexity; merchants will not understand the distinction.

**Recommended: Approach B (Hard Reset) with a confirmation modal.**

The user explicitly requested "delete all the data associated to that bundle and re-create the bundle again." A single "Sync Bundle" button that does a full hard reset covers both the light sync and structural recovery use cases. A confirmation modal prevents accidental use.

## Functional Requirements (FRs)

- **FR-01:** A "Sync Bundle" button must appear on both the full-page bundle configure page and the product-page bundle configure page.
- **FR-02:** Clicking the button opens a confirmation modal warning that all Shopify-side data will be deleted and re-created.
- **FR-03:** On confirm, the action deletes all DB child records (steps, pricing) for the bundle, then re-creates them from the submitted form state.
- **FR-04:** On confirm, the action deletes the Shopify product (product-page) or Shopify page (full-page) associated with the bundle, then re-creates it.
- **FR-05:** On confirm, all three metafield sync operations are re-run: bundle product metafields, component product metafields, standard metafields.
- **FR-06:** The bundle primary ID (`Bundle.id`) is preserved throughout.
- **FR-07:** On success, a toast notification confirms the sync completed.
- **FR-08:** On failure, a toast notification shows the error; partial state is logged.
- **FR-09:** The button is disabled while a sync is in progress.

## Non-Functional Requirements (NFRs)

- **NFR-01: Atomicity** — If Shopify deletion succeeds but re-creation fails, the DB must reflect the incomplete state (not silently show stale data).
- **NFR-02: Safety** — The confirmation modal must require an explicit user action (button click) before any destructive operation begins.
- **NFR-03: Analytics preservation** — `BundleAnalytics` records must NOT be deleted (they are historical; cascade delete must be avoided).
- **NFR-04: Performance** — The operation should complete within 30 seconds for a bundle with up to 10 steps and 50 products.
- **NFR-05: Idempotency** — Running the sync twice in a row must produce the same result.

## User Stories (high level)

- As a **developer**, I want to sync a bundle after a breaking app update so that the storefront widget works correctly without manually re-creating the bundle.
- As a **merchant**, I want to recover a broken bundle whose Shopify product was accidentally deleted so that my customers can use it again.
- As a **developer**, I want the bundle's page URL and product handle to remain the same after sync so that external links don't break.

## Scope

### In Scope
- Sync button on full-page bundle configure page
- Sync button on product-page bundle configure page
- Confirmation modal before destructive action
- Delete + re-create: Shopify product/page, all metafields, DB steps and pricing
- Preserve: `Bundle.id`, `Bundle.shopId`, `BundleAnalytics` records
- Toast notifications for success and failure

### Out of Scope
- Bulk sync across all bundles
- Sync scheduling / automation
- Sync history / audit log
- Cart transform re-registration (shop-wide, not per-bundle)
- Storefront theme template re-installation

## Risks

| Risk | Likelihood | Mitigation |
|------|-----------|-----------|
| Shopify rate limiting during metafield sync | Medium | Use batching (already implemented in sync services) |
| Partial failure leaves bundle in broken state | Medium | Log partial state; surface clear error to user |
| `BundleAnalytics` accidentally deleted via cascade | Low | Ensure analytics are excluded from the delete operation |
| New Shopify product GID breaks cart transform | Low | Cart transform reads metafields from component variants, not product GID directly |
| Merchant accidentally clicks Sync and loses data | Medium | Require confirmation modal with explicit destructive warning |
