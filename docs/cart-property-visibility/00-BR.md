# Business Requirement: Cart Property Visibility Fix

## Executive Summary
Merchants using custom Shopify themes see raw internal bundle attributes (`_bundle_name`, `_is_bundle_parent`, etc.) exposed on their cart page. We need to surface a clear, copy-pasteable Liquid snippet inside the app so merchants can fix this in under 2 minutes.

## Problem Statement
Our Cart Transform sets 8 `_`-prefixed line item attributes on the bundle parent (used by the Checkout UI Extension). Shopify convention says `_`-prefixed properties should be hidden from customers — Dawn and most modern themes implement this natively. Merchants on custom themes that skip this convention see all 8 attributes rendered verbatim on the cart page, breaking the checkout UX and exposing raw internal data to customers.

## Research Findings
- **Approach A: App Embed JS block** — DOM mutation / MutationObserver to hide elements by text content. Pros: zero merchant action. Cons: theme-specific DOM selectors, fragile on AJAX cart drawers, maintenance burden. **Rejected (previously reverted).**
- **Approach B: Liquid snippet (theme-side fix)** — Merchant adds a one-line `unless` guard to their cart template's property loop. Pros: authoritative, Shopify-recommended, permanent, zero JS overhead. Cons: requires merchant to edit theme. **Recommended.**

## Functional Requirements
- FR-01: Display the exact Liquid snippet merchants need to add to their cart template
- FR-02: Provide a copy-to-clipboard button for the snippet
- FR-03: Include step-by-step instructions (3 steps max)
- FR-04: Note that Dawn and most modern themes don't need this fix
- FR-05: Surface this on the dashboard — no additional navigation required

## Non-Functional Requirements
- NFR-01: No new route needed — dashboard card is sufficient
- NFR-02: No backend changes — pure UI component
- NFR-03: Works in Shopify embedded app iframe (clipboard API)

## User Stories
- As a merchant on a custom theme, I want clear instructions to hide internal bundle properties from my cart page, so my customers don't see technical noise.

## Scope
### In Scope
- `CartPropertyFixCard` component on the dashboard
- Copy-to-clipboard with visual confirmation
- Step-by-step instructions + snippet

### Out of Scope
- Automatic theme patching
- App Embed JS fallback
- Detection of whether merchant's theme needs the fix
