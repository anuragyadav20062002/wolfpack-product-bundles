# Business Requirement: Events Page

## Executive Summary
Add a dedicated Events page to the app that surfaces release notes and how-to guides as accordion-style cards, improving discoverability of new features and self-serve support content.

## Problem Statement
Merchants currently have no in-app way to discover recent app changes or find fix guides (like the Cart Property Display Fix). Dashboard real estate is limited. A dedicated page gives structured, scalable space for both.

## Functional Requirements
- FR-01: Events page accessible from main NavMenu
- FR-02: Two sections — "Latest Events" and "FAQs & Tutorials"
- FR-03: Each item is an accordion — collapsed shows title + subtitle; expanded shows full content
- FR-04: CartPropertyFixCard content migrates from dashboard into FAQs & Tutorials
- FR-05: Dashboard removes CartPropertyFixCard after migration
- FR-06: Smooth expand/collapse animation

## Non-Functional Requirements
- NFR-01: No DB or loader needed — static content, pure UI
- NFR-02: No tests required (Polaris UI TDD exception)
- NFR-03: Consistent with existing Polaris design patterns

## Scope
### In Scope
- New route `app.events.tsx`
- `AccordionItem` reusable component
- Nav link in `app.tsx`
- Remove CartPropertyFixCard from dashboard

### Out of Scope
- CMS-driven content
- Pagination
- Search/filter
