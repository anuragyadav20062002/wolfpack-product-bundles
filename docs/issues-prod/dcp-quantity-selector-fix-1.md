## Overview

Customer (Bay & Co.) configured the quantity selector in DCP with white background
(`#ffffff`) and dark text (`#111827`). The setting was ignored on the storefront â€”
quantity selector still rendered with the hardcoded default grey (`#5f5d5d`).

Also: the DCP preview for landing page bundles must match the actual storefront
implementation exactly.

## Root Cause

`.inline-quantity-controls` in `bundle-widget.css` had hardcoded `background: #5f5d5d`
with no CSS variable consumption and no text-color or border-radius rules.
The CSS variables `--bundle-quantity-selector-bg`, `--bundle-quantity-selector-text-color`,
and `--bundle-quantity-selector-border-radius` were correctly generated by the CSS
generator but never consumed by this selector.

## Progress Log

### 2026-04-12 - Root Cause Found

Audit via Chrome DevTools on customer store (`baygifts.myshopify.com`).
DCP settings confirmed: bg `#ffffff`, text `#111827`, border-radius 12px.

Root cause: `bundle-widget.css` `.inline-quantity-controls` rule:
```css
/* Before */
.inline-quantity-controls {
  background: #5f5d5d;
}
```

No variable consumption. The CSS generator correctly outputs
`--bundle-quantity-selector-bg: #ffffff` but no rule reads it for this selector.

### 2026-04-12 - Fix Applied

Full replacement:
```css
/* After */
.inline-quantity-controls {
  display: flex;
  align-items: center;
  background: var(--bundle-quantity-selector-bg, #5f5d5d);
  color: var(--bundle-quantity-selector-text-color, #ffffff);
  border-radius: var(--bundle-quantity-selector-border-radius, 8px);
  overflow: hidden;
}

.inline-qty-btn {
  background: transparent;
  color: inherit;
  border: none;
  cursor: pointer;
  padding: 0 10px;
  font-size: var(--bundle-quantity-selector-font-size, 16px);
  font-weight: bold;
  line-height: 1;
}

.inline-qty-display {
  color: inherit;
  font-size: var(--bundle-quantity-selector-font-size, 16px);
  font-weight: 500;
  min-width: 24px;
  text-align: center;
}
```
