# Business Requirement: Single Embed Bundle Architecture

## Executive Summary

Wolfpack must use one merchant-facing Theme Editor app embed for both full-page bundles and product-page bundles, while retaining deterministic app-block placement for bundle builders and upsell widgets. The architecture should match the verified competitor behavior without relying on competitor private configuration.

## Problem Statement

The current extension exposes separate full-page and product-page app embeds. This creates avoidable merchant setup complexity and diverges from the verified single-embed competitor model.

## Requirements

- Show one Theme Editor app embed named `Wolfpack Bundle`.
- Render full-page bundles through the verified app proxy path `/apps/product-bundles/wpb/:bundleId`.
- Keep explicit app blocks for exact placement of the bundle builder and upsell surfaces.
- Make existing bundles move through a sync-required cutover, with no backwards-compatibility shims.
- Keep storefront-visible controls compatible with DCP/design settings.

## Success Criteria

- Theme Editor App Embeds shows one Wolfpack embed.
- FPB configure shows `Bundle Visibility` with only `Bundle Widget`.
- PPB configure shows `Bundle Visibility` with `Bundle Widget` and `Bundle Embed`.
- FPB public links open `/apps/product-bundles/wpb/:bundleId`.
- PPB direct builder placement uses a real app block.
- Deploy is manual through the approved npm deploy scripts only.
