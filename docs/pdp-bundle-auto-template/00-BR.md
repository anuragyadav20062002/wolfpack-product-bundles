# Business Requirement: PDP Bundle Auto-Template

## Executive Summary

When a merchant creates a product-page bundle, the app must automatically create a
custom Shopify product template (`templates/product.product-page-bundle.json`) in the active theme
and apply it to the bundle's linked product — exactly as it already does for full-page
bundles — so no Theme Editor interaction is required.

## Problem Statement

**Who:** Merchants setting up product-page bundles (PDP bundles).

**Pain point:** After configuring a bundle and selecting a linked product, the widget does
not appear on that product's storefront page. The merchant must manually go to the Shopify
Theme Editor, navigate to the product page template, add the `bundle-product-page` app block,
configure it with the correct bundle ID, then save. This is a multi-step, opaque flow that
most merchants fail to complete correctly.

**Contrast:** Full-page bundles already receive a custom page + template automatically
(via `createFullPageBundle()`). Product-page bundles should enjoy the same zero-friction
installation experience.

**Competitive context (Skai Lama research):**
Skai Lama's "Easy Bundle Builder" product pages (e.g. `ultimate-soft-toy-bundle-new`,
`swag-create-your-look-new`) show the bundle widget rendered inline on the standard
product page — in the right column, immediately after pricing — with no visible Theme
Editor setup step required. Their template places the widget block automatically when
a bundle is linked to a product.

## Research Findings

### Approach A — Manual Theme Editor (current state)
Merchant adds the `bundle-product-page` block manually.
- Pros: Zero code change.
- Cons: High friction, high support burden, most merchants never complete it.
- **Verdict: ❌ Status quo — rejected.**

### Approach B — Inject block into existing product template
Modify the active theme's `templates/product.json` in-place to add our block.
- Pros: No template suffix needed; works on the merchant's existing template.
- Cons: Mutating shared theme files is destructive. Affects ALL products, not just bundle
  products. Reverting cleanly is nearly impossible. Violates Shopify guidelines.
- **Verdict: ❌ Rejected — too invasive.**

### Approach C — Custom product template with templateSuffix (recommended)
Create `templates/product.product-page-bundle.json` in the active theme with an `apps`
section containing the `bundle-product-page` block. Set `templateSuffix: 'product-page-bundle'`
on the linked Shopify product so only that product uses this template.
- Pros: Matches Shopify's recommended pattern. Exactly mirrors what we already do for
  full-page bundles. Isolated — only affects products with `templateSuffix: 'product-page-bundle'`.
  Template creation is idempotent (second save is a no-op). Fully reversible.
- Cons: Requires one Admin REST API write to theme assets + one GraphQL mutation on the product.
- **Verdict: ✅ Recommended.**

### Approach D — App embed block (Online Store 2.0 app embed)
Use an app embed to inject the widget via JavaScript globally.
- Pros: No per-product setup.
- Cons: Renders on every product page; requires per-product show/hide logic in JS;
  performance impact; does not match Skai Lama's inline-section pattern.
- **Verdict: ❌ Rejected — wrong granularity.**

## Functional Requirements

- **FR-01:** When a product-page bundle is saved with a linked Shopify product, the app
  must ensure `templates/product.product-page-bundle.json` exists in the merchant's active theme.
- **FR-02:** The template must include an `apps` section with the `bundle-product-page`
  block embedded using the extension UID from `shopify.extension.toml`.
- **FR-03:** After the template is ensured, the linked Shopify product must be updated
  with `templateSuffix: 'product-page-bundle'`.
- **FR-04:** The operation must be idempotent — saving the same bundle multiple times
  must not create duplicate templates or fail.
- **FR-05:** If the template file already exists in the theme, skip writing it and proceed
  directly to applying the suffix to the product.
- **FR-06:** If the template write fails (transient API error), the bundle save must still
  succeed. The failure is logged as a warning (non-fatal), matching full-page bundle behaviour.
- **FR-07:** When Sync Bundle is triggered for a product-page bundle, the same
  template-ensure + suffix-apply flow must run as part of the sync.
- **FR-08:** The base template is derived from `templates/product.json` in the active theme
  so the merchant's existing product section (images, title, price, description) is preserved.

## Non-Functional Requirements

- **NFR-01 Performance:** Template write adds at most 2 REST API calls (list assets + PUT).
  These run in parallel with existing metafield operations where possible.
- **NFR-02 Backward compatibility:** Existing bundles already live on product pages are
  unaffected. The template only applies to products explicitly set as bundle containers.
- **NFR-03 Idempotency:** `themeAssetExists()` check prevents redundant writes.
- **NFR-04 Shopify guidelines:** Only one named template is created (`product.product-page-bundle.json`).
  No existing theme files are mutated.

## User Stories (high level)

- As a merchant, I want the bundle widget to appear on my product page automatically
  when I save a bundle, so I don't have to touch the Theme Editor.
- As a merchant, saving the bundle configuration a second time should not break anything
  that was already working.
- As a merchant, if the auto-setup fails silently, I still want my bundle data saved so I
  can recover manually.

## Scope

### In Scope
- Creating `templates/product.product-page-bundle.json` on bundle save
- Setting `templateSuffix: 'product-page-bundle'` on the linked Shopify product
- Running the same flow in Sync Bundle for product-page bundles
- Idempotency checks (skip if template already exists)
- Non-fatal failure handling (warn + continue)

### Out of Scope
- Reverting `templateSuffix` when a bundle is deleted or unlinked (future work)
- Customising the product template layout beyond our standard block placement
- Modifying the widget block's Liquid markup
- Product-page bundles without a linked Shopify product (no-op)

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Theme REST API rate-limit during bulk bundle saves | Low | Existing retry/error handling; non-fatal |
| Merchant's `templates/product.json` has non-standard structure | Medium | Fallback to minimal template if parse fails |
| Extension UID changes after `shopify app deploy --reset` | Very Low | Read UID from TOML at runtime (not hardcoded) |
| Product template write modifies a theme managed by a third-party app | Low | Template key is unique (`product.product-page-bundle.json`); no conflict |
