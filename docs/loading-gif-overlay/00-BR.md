# Business Requirement: Loading GIF Overlay for Bundle Widgets

## Executive Summary

Merchants need a branded loading experience during three critical wait moments in bundle widgets (initial load, step transitions, Add to Cart). A merchant-configurable GIF — or a clean fallback spinner — should appear over the widget area during each moment without blocking the rest of the page.

## Problem Statement

**Who experiences it:** Shoppers using both full-page and product-page bundle widgets.

**Pain points:**
- Initial bundle load shows a raw skeleton animation — no branded experience.
- Step transitions in the full-page widget navigate instantly to a blank/loading product grid with no feedback.
- Clicking "Add to Cart" / "Add Bundle To Cart" gives no immediate visual confirmation that the request is processing; users may click multiple times.

**Merchant pain:** No way to customise the loading experience to match their brand.

## Research Findings

### Approach A: Full-page overlay (blocks entire viewport)
Use a fixed-position overlay covering `100vw × 100vh`.
- Pros: Simple implementation.
- Cons: Breaks the requirement "must not block the entire page". Poor UX for pages with other content.
- **Verdict: ❌ Rejected**

### Approach B: Widget-scoped overlay (absolute within container)
Use `position: absolute; inset: 0` inside the widget container (`position: relative`).
- Pros: Scoped to the bundle area; rest of page remains interactive; respects z-index hierarchy.
- Cons: Container must have `position: relative` (easy to ensure in widget JS).
- **Verdict: ✅ Recommended**

### Approach C: Inline skeleton replacement
Swap the skeleton cards for a centered GIF/spinner.
- Pros: Feels integrated.
- Cons: Requires redesigning skeleton logic; complex state tracking.
- **Verdict: ❌ Rejected** — overlay is simpler and more reliable.

**Recommended:** Approach B — widget-scoped absolute overlay.

## Functional Requirements (FRs)

- FR-01: Show a loading overlay over the widget area during initial bundle data fetch.
- FR-02: Show a loading overlay during full-page step transitions (while `loadStepProducts()` runs).
- FR-03: Show a loading overlay while the Add to Cart / Add Bundle To Cart fetch is in progress; disable the button.
- FR-04: Merchants configure the GIF URL via the "Images & GIFs" section in both configure pages.
- FR-05: If no GIF is configured, display a clean CSS spinner as fallback.
- FR-06: The GIF must be capped at 120×120 px display size.
- FR-07: The overlay must fade in and out (≈200 ms transition).
- FR-08: The overlay must be scoped — it must not cover the browser chrome, navigation, or other page sections.
- FR-09: The product-page configure page must gain an "Images & GIFs" section to expose the GIF upload.
- FR-10: `loadingGif` must be persisted to DB, synced to the `bundle_ui_config` metafield, and returned by the bundle data API.

## Non-Functional Requirements (NFRs)

- NFR-01: Zero regression on existing widget behaviour (init, step navigation, add to cart).
- NFR-02: No additional network requests — GIF URL is already in bundle data payload.
- NFR-03: Graceful degradation — if no GIF configured, spinner renders from pure CSS.
- NFR-04: Backward compatible — existing bundles without `loadingGif` default to spinner.
- NFR-05: CSS added to shared `bundle-widget.css` so both widget types benefit.

## User Stories (high level)

- As a merchant, I want to upload a branded loading GIF per bundle so shoppers see my brand during wait states.
- As a shopper, I want clear visual feedback when a step is loading or my cart is being updated.
- As a developer, I want the overlay to be a reusable, CSS-only component driven by a single JS method pair (`showLoadingOverlay` / `hideLoadingOverlay`).

## Scope

### In Scope
- `loadingGif` DB column on Bundle model
- `loadingGif` in metafield types and builder
- `loadingGif` in bundle data API response
- Images & GIFs section on **product-page** configure (new)
- Images & GIFs section on **full-page** configure (add loading GIF card)
- CSS overlay component in `bundle-widget.css`
- JS overlay methods in both widget files + rebuild bundles
- Issue tracking file + commit

### Out of Scope
- Custom overlay background colour or opacity per merchant
- Loading overlay for variant selector modal within steps
- Video support (GIF only for now)
- Analytics tracking of overlay display times
