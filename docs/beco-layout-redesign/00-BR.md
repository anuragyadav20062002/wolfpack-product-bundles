# Business Requirement: Beco BYOB Expandable Floating Footer

## Executive Summary

Replace the current multi-section floating footer on full-page bundles with a compact, Beco-inspired
sticky footer bar that expands upward to reveal selected products (with remove actions) and a
deal-unlock callout — eliminating clutter and improving mobile UX.

## Problem Statement

The current full-page bundle footer is noisy: it stacks a progress section, a horizontal scrollable
product tiles row, and a back/next navigation row. On mobile this consumes excessive viewport space
and confuses the primary CTA. Merchants like Beco use a compact bar with a single expand toggle that
keeps the viewport clean while still giving customers full visibility of what's in their box.

## Research Findings

### Beco BYOB Live Reference
URL: https://www.letsbeco.com/pages/BYOB_13May/
Analysed live via Chrome DevTools — expanded footer captured at height 293px (fixed positioned).

**Collapsed footer bar (90px):**
- Left zone: small thumbnail strip (≤3 circular/square images, ~36px each, overlapping)
- Centre zone: "X/Y Products ∨" toggle button + "Total: ₹499  38% OFF" pill
- Right zone: primary CTA "BUY BUNDLE" (black, full-height button)

**Expanded panel (opens above bar):**
- Green deal-callout banner: "🎉 Best Deal Unlocked! Get X products at ₹YYY"
- `<ul>` of selected products — each row: 48×48 thumbnail | product name | ₹price × qty | 🗑️ trash icon
- Transparent backdrop div — click to collapse
- Chevron flips ∨ → ∧ when open

**HTML structure (reconstructed from DevTools):**
```
.buy-x-at-y-cart (fixed, bottom:0)
  .bxay-cart-callout__container  (deal banner, hidden when collapsed)
  button.buy-x-at-y-cart__backdrop  (transparent, click to close)
  .buy-x-at-y-cart__lines  (product list, hidden when collapsed)
    ul > li × N
      img 48×48 | .product-info (name + price×qty) | button.trash
  .buy-x-at-y-cart__bar  (always visible)
    .thumbnails-strip
    button.toggle  "X/Y Products ∨/∧"
    .total-area    "Total: ₹499  38% OFF"
    button.cta     "BUY BUNDLE"
```

### Approach A: Full rewrite of `renderFooter()`
Pros: Clean implementation, matches Beco exactly, no legacy cruft.
Cons: Breaks existing DCP CSS variables for footer (must migrate/alias them).
**Recommended ✅**

### Approach B: Overlay panel on top of existing footer
Pros: Less code change.
Cons: Conflicting z-index, stale animation state, duplicated DOM nodes.
**Rejected ❌**

## Functional Requirements

- FR-01: Footer bar is always visible at the bottom of the viewport when at least 1 product is selected.
- FR-02: Footer shows a thumbnail strip of selected product images (up to 3, with "+N" overflow badge).
- FR-03: Footer shows selected count "X/Y Products" with chevron toggle button.
- FR-04: Footer shows total price and discount % badge.
- FR-05: Footer primary CTA ("Add to Cart" / "BUY BUNDLE") submits the bundle.
- FR-06: Clicking the toggle opens an upward-expanding panel listing all selected products.
- FR-07: Each row in the expanded panel shows: thumbnail, name, price × qty, trash (remove) icon.
- FR-08: Clicking trash removes the product from the bundle (calls existing deselect logic).
- FR-09: A green deal-callout banner appears at the top of the expanded panel when the bundle goal is met.
- FR-10: Clicking the backdrop or toggle button again collapses the panel.
- FR-11: The footer must respect the `footer_side` layout (hide when sidebar layout is active).
- FR-12: Back navigation between steps is preserved (either in the bar or above it).

## Non-Functional Requirements

- NFR-01: No regressions on footer_side layout or product-page widget footer.
- NFR-02: All new CSS variables follow existing `--bundle-full-page-footer-*` naming convention.
- NFR-03: Mobile-first: footer bar ≤ 80px tall; expanded panel max-height 60vh with scroll.
- NFR-04: Smooth CSS transition for expand/collapse (max-height + opacity, ~250ms).
- NFR-05: DCP settings for footer background, text, CTA button colours must still work.

## User Stories (high level)

- As a shopper, I want to see a compact footer so I can browse products without the footer dominating my screen.
- As a shopper, I want to tap "X/Y Products" to review everything I've added before buying.
- As a shopper, I want to remove a product from the expanded panel without losing my place on the page.
- As a merchant, I want DCP colour controls for the footer bar and CTA button to still apply.

## Scope

### In Scope
- Full rewrite of `renderFooter()` in `bundle-widget-full-page.js`
- New CSS in `bundle-widget-full-page.css` (new selectors, existing selectors kept for DCP vars)
- DCP section for footer in the admin app (new subsection: "Expandable Footer")
- Preserve Back/Next step navigation (embed Back button in the bar)

### Out of Scope
- Product-page widget footer (separate component)
- footer_side layout sidebar panel (unchanged)
- Server-side changes

## Risks

| Risk | Likelihood | Mitigation |
|------|-----------|-----------|
| DCP CSS variable names conflict with new selectors | Medium | Alias old vars to new component classes |
| Expand animation janky on low-end Android | Low | Use `max-height` transition + `will-change: max-height` |
| Thumb-strip overflow badge ("+ N") confusing on desktop | Low | Cap strip at 5 on desktop, 3 on mobile |
