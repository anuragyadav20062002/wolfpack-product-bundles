# Business Requirement: Product-Page Bundle Widget — Bottom-Sheet Redesign

**Feature Name:** skai-lama-bottom-sheet-redesign
**Stage:** 1 — Business Requirement
**Author:** Feature Pipeline
**Date:** 2026-03-17

---

## Executive Summary

Our product-page bundle widget uses a vertical-accordion/inline UX that lags behind competitors like Skai Lama (Easy Bundle Builder), who ship a bottom-sheet modal with auto-step progression that guides customers through selection in a polished, conversion-optimised flow. Adopting this pattern will improve bundle completion rates and bring feature parity with the market leader.

---

## Problem Statement

**Who experiences it:** Merchants who embed our product-page bundle widget on their Shopify product pages. Their shoppers (end customers) are the direct sufferers.

**Pain points:**

1. **Low conversion compared to Skai Lama stores.** Our inline accordion exposes all steps simultaneously and requires customers to scroll. Skai Lama's step-by-step bottom-sheet guides customers one category at a time, reducing cognitive load and increasing completion.

2. **No visual "empty slot" affordance.** When a shopper lands on the page, our widget shows labeled boxes with a "+" icon — shoppers don't immediately understand they are clickable slots for selecting products. Skai Lama shows category images in dashed-border slots, making the CTA visually obvious.

3. **No auto-progression.** After selecting a product in a step, shoppers must manually click "Next" or switch tabs. Skai Lama automatically advances to the next category the moment a step's quantity condition is met — 40-60% of friction is removed in the selection flow.

4. **No discount badge on inline cards.** Free-gift steps and heavily discounted steps have no visual call-out on the inline widget. Skai Lama shows a "100% off" pill badge on filled cards for these steps, reinforcing the value proposition.

5. **No default/pre-selected product support.** Merchants cannot pre-select a product for a step (e.g., "includes 1 bottle of X"). Skai Lama supports default products that are always filled and non-removable by the customer.

6. **Modal animation is full-screen, not bottom-sheet.** Our modal overlays the full viewport with a center-anchored dialog. Skai Lama's bottom-sheet slide-up (400ms ease-in-out) feels native and premium. On mobile it fills ~85% of the screen without obscuring the product page header.

## Research Findings

All findings based on hands-on Chrome DevTools analysis of:
- `skailama-demo.myshopify.com/products/swag®-create-your-look-new`
- `skailama-demo.myshopify.com/products/ultimate-soft-toy-bundle-new`

Documented in full in `docs/skai-lama-analysis/findings.md` with 19 screenshots.

### Approach A — Bottom-Sheet Overlay Modal (Skai Lama pattern)
**Description:** A `position: fixed; bottom: 0` panel that slides up when a customer clicks an empty slot in the inline widget. The panel contains horizontal category tabs + product grid. Auto-advances tabs when step condition is met. Closes when all steps are complete.

**Pros:**
- Matches industry-standard pattern (Skai Lama, multiple other bundle apps)
- Reduces scrolling — customer selects products in a focused overlay
- Auto-step progression dramatically reduces friction
- Bottom-sheet feel is native on mobile (mirrors native iOS/Android patterns)
- Modal does not navigate away from product page — customers feel safe

**Cons:**
- Significant widget JS + CSS rewrite (~1,200–1,500 lines of net-new code)
- Existing step accordion must be replaced by inline "slot" cards
- DCP needs new controls + preview sections

**Verdict: ✅ Recommended** — this is the right UX architecture and matches the market leader directly.

### Approach B — Inline Wizard (step-by-step scroll-jacking)
**Description:** Keep existing inline layout but show one step at a time with a "Next" CTA. Scroll to next step automatically.

**Pros:**
- Less code change — keeps existing DOM structure
- No modal to build

**Cons:**
- Requires page scroll control — risks breaking theme scroll behavior
- Does not feel "clean" compared to modal isolation
- Still inferior UX to the bottom-sheet pattern
- Harder to do on mobile

**Verdict: ❌ Rejected** — scroll-jacking creates unpredictable behavior in themes and is a worse UX outcome.

### Approach C — Full-page-style embedded inline (no modal)
**Description:** Render all products for all steps inline simultaneously (like our full-page widget, but on the product page).

**Pros:**
- Simpler JS — just render everything

**Cons:**
- Pollutes the product page with a huge content block — bad for merchant themes
- Does not match any competitor pattern for product-page placement
- Our full-page widget already serves this need for dedicated bundle pages

**Verdict: ❌ Rejected** — not appropriate for product-page placement.

## Functional Requirements (FRs)

- **FR-01:** Inline widget shows one card slot per step; each slot displays the step's category representative image + step name + dashed border (empty state).
- **FR-02:** Clicking an empty slot or filled card opens the bottom-sheet modal anchored to `bottom: 0` of the viewport.
- **FR-03:** Modal slides up with CSS `height 0 → max-height` transition, 400ms ease-in-out, `border-radius: 15px 15px 0 0`.
- **FR-04:** A semi-transparent overlay (`rgba(30,30,30,0.5)`) covers the page behind the modal.
- **FR-05:** Modal header contains horizontal scrollable category tabs — one per non-default step. Active tab = brand color background + white text.
- **FR-06:** Modal body shows a product grid: 5 columns desktop, 2 columns mobile. Selected product shows dark border + shadow + checkmark icon.
- **FR-07:** When a step's quantity condition is met (e.g., 1 product selected for "equal_to 1" step), modal auto-advances to the next non-complete step tab.
- **FR-08:** When all steps are complete, modal closes automatically.
- **FR-09:** Filled inline card shows product image + title + variant + remove button (×). Remove button is NOT shown for default/pre-selected products.
- **FR-10:** Steps marked as "default" are pre-filled from page load, have no modal tab, and cannot be removed by the customer.
- **FR-11:** Steps with a 100% discount (free gift) or configured step-level discount badge show a discount pill (e.g., "100% off") on the filled inline card.
- **FR-12:** "Add Bundle to Cart" button is disabled (gray) until all steps are complete; enabled state shows brand primary color + embedded price + discount badge.
- **FR-13:** On mobile, modal close button is a chevron-down (↓) icon; on desktop it is an X.
- **FR-14:** All new visual settings (modal border-radius, overlay opacity, empty-slot border style/color, discount badge color) are exposed as DCP controls and preview-able in the DCP preview panel.
- **FR-15:** Existing merchants who have not opted into the new layout continue to see the current widget behavior (backward-compatible via a DCP toggle or automatic version detection).

## Non-Functional Requirements (NFRs)

- **NFR-01 Performance:** Modal products must still be loaded lazily (on first open of that step tab). Preloading of next step happens in background while modal is open. No regression in time-to-interactive.
- **NFR-02 Backward Compatibility:** Existing `bundle_ui_config` JSON metafield schema must not require a migration. New fields (e.g., `defaultProduct`, `discountBadgeEnabled`) must be optional with safe defaults.
- **NFR-03 Accessibility:** Modal must trap focus when open. Overlay click + Escape key must close the modal.
- **NFR-04 Mobile-first:** Modal height must be responsive (85% viewport on mobile, capped at ~765px on desktop). Category tabs must scroll horizontally.
- **NFR-05 Theme compatibility:** `position: fixed` modal must not be clipped by ancestor `overflow: hidden` elements. Z-index must be sufficiently high (9999999) to clear Shopify theme headers.
- **NFR-06 Widget size:** Net increase in bundled JS size should be < 30KB (minified). The bottom-sheet replaces most of the existing modal JS, not additive to it.
- **NFR-07 DCP preview fidelity:** The new empty-slot and modal states must be previewable in the DCP without requiring a live storefront.

## User Stories (High Level)

- As a **shopper**, I want to see a clean set of "select your item" slots when I land on a bundle product page, so that I immediately understand what I need to choose.
- As a **shopper**, I want a bottom-sheet to slide up when I click a slot, so that I can pick a product from that category without leaving the page context.
- As a **shopper**, I want the widget to automatically move me to the next selection step after I choose a product, so that I don't have to manually navigate.
- As a **shopper**, I want to see "FREE" or "100% off" highlighted on gift items in my bundle, so that the value proposition is immediately clear.
- As a **merchant**, I want to pre-select default products for certain steps (e.g., a mandatory accessory), so that shoppers can't accidentally build an incomplete bundle.
- As a **merchant**, I want to customise the modal border radius, overlay colour, empty-slot border style, and discount badge colour in the DCP, so that the widget fits my store's brand.

## Scope

### In Scope
- Bottom-sheet modal replacing existing center-anchored modal (product-page widget only)
- Auto-step progression when step condition is met
- Empty-state slot cards in inline widget (with category image + dashed border)
- Default/pre-selected product step support
- Discount badge pill on filled inline cards for free-gift / discounted steps
- DCP controls + preview for all new settings
- Mobile-responsive modal (85vh height, 2-col product grid, chevron-down close)
- Backward-compatible `bundle_ui_config` JSON schema additions

### Out of Scope
- Full-page bundle widget (separate codebase, already improved in beco-design-alignment-1)
- Cart transform / pricing logic (unchanged)
- Checkout UI extension (unchanged)
- Shopify admin product configuration changes
- Spring-physics toast animation (nice-to-have, deferred)
- Skai Lama's star rating / review counts (no native Shopify ratings API)
- Multi-bundle-per-page selection UI

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| `position: fixed` modal clipped by theme `overflow: hidden` | Medium | Append modal to `document.body`; test with Dawn, Debut, Prestige themes |
| Merchants rely on existing accordion layout — disruptive change | Low | Provide a DCP toggle "Widget style: Classic / Bottom-Sheet" — default new installs to Bottom-Sheet |
| CSS z-index conflicts with theme nav/cart overlays | Medium | Use z-index: 9999999; test against sticky header themes |
| Auto-step progression confuses customers who want to review all products | Low | Allow clicking any tab to navigate back; only auto-advance, never auto-close incomplete |
| Default product data not in metafield for older bundles | Low | `defaultProduct` field is optional — existing bundles have no default products, widget ignores absence |
| Widget bundle size increase > 30KB | Low | Bottom-sheet replaces existing modal — net delta expected to be +8–12KB |
