# Business Requirement: Promo Banner Background Image from Shopify Files

**Feature ID:** promo-banner-bg-image
**Created:** 2026-02-20
**Status:** BR Complete → Handoff to PO

---

## Executive Summary

Merchants want to use a custom image (already uploaded to their Shopify store's Content → Files) as the promo banner's background in the full-page bundle widget, instead of being limited to a solid background colour. This gives the banner a richer, on-brand visual that matches the rest of the storefront.

---

## Problem Statement

The promo banner in the full-page bundle widget currently supports only a solid background colour (`promoBannerBgColor`). Merchants who want a branded look — product photography, gradient overlays, lifestyle imagery — have no way to set a background image. The images they've already uploaded to Shopify's Content → Files are inaccessible from within the app's Design Control Panel (DCP).

**Who experiences this:** Merchants using the full-page bundle widget who want their bundle landing pages to look premium rather than plain.

---

## Research Findings

### Current system (baseline)
- DCP stores 11 promo banner fields in `DesignSettings.promoBannerSettings` (Json column, Postgres).
- `css-variables-generator.ts` emits `--bundle-promo-banner-bg` from `promoBannerBgColor`.
- Widget CSS: `.promo-banner { background: var(--bundle-promo-banner-bg, #F3F4F6); }` — solid colour only.
- No image fetching, no file upload, no `background-image` CSS anywhere in the pipeline.

### Approach A — Upload direct from DCP (file upload within app)
Merchant uploads a new image directly via the app UI using Shopify's `stagedUploadsCreate` + `fileCreate` Admin GraphQL mutations.

**Pros:** All-in-one flow inside the app.
**Cons:** Requires building a full upload pipeline (staged upload → polling for processing → CDN URL). Duplicates files already in the store. Higher engineering cost. Shopify rate-limits staged uploads.

**Verdict:** ❌ Over-engineered for this use case.

### Approach B — File picker from Content → Files (recommended)
Merchant picks an image they have already uploaded to Content → Files using a searchable picker in DCP. The app queries the Shopify Admin GraphQL `files` query, displays thumbnails, and stores the selected CDN URL.

**Pros:**
- Zero new upload infrastructure — uses what merchants already have.
- Shopify Files CDN URLs are publicly accessible, no CORS issues in the storefront widget.
- Simple read-only API query (`files(first: 25, query: "media_type:Image")`).
- Fits naturally into the existing DCP settings flow (store URL string in `promoBannerSettings` Json).
- No new DB migration needed — the existing `promoBannerSettings Json` column absorbs the new field.

**Cons:** Merchant must upload their image to Shopify Files separately before using this feature (reasonable — they likely already do this).

**Verdict:** ✅ Recommended.

### Approach C — External URL input
Merchant pastes an arbitrary image URL.

**Pros:** Trivial to implement.
**Cons:** CORS risk for cross-origin images, no thumbnail preview, poor UX, security concerns, breaks if external URL changes.

**Verdict:** ❌ Rejected — unreliable and insecure.

## Functional Requirements

- **FR-01:** The DCP Promo Banner settings panel (full-page only) must include a "Background Image" section.
- **FR-02:** The section must show a searchable/scrollable picker of images from the store's Content → Files, fetched via Shopify Admin GraphQL `files` query.
- **FR-03:** Each file in the picker must show a thumbnail, filename, and date added.
- **FR-04:** The merchant can select one image or clear the selection (revert to solid colour).
- **FR-05:** When an image is selected, it overrides `promoBannerBgColor` as the banner background. When cleared, the solid colour is used again.
- **FR-06:** The selected image URL must persist in `DesignSettings.promoBannerSettings` and survive page reloads.
- **FR-07:** The CSS variable pipeline must emit a new `--bundle-promo-banner-bg-image` variable pointing to the CDN URL.
- **FR-08:** The widget CSS must apply `background-image: url(...)` with `background-size: cover; background-position: center` when the variable is set.
- **FR-09:** When a background image is active, the solid colour acts as a fallback (for slow network / image load failure).

---

## Non-Functional Requirements

- **NFR-01 Backward compatibility:** Existing `DesignSettings` rows with no image URL must continue to render with solid colour. No data migration needed.
- **NFR-02 Performance:** The file picker query must use pagination (`first: 25`) and support load-more. Images in the picker must use Shopify CDN thumbnail parameters to avoid loading full-resolution images in the admin UI.
- **NFR-03 Security:** Only read access to the store's own files. No cross-origin image URLs are stored or served.
- **NFR-04 Widget size:** The new CSS variable adds ~1 line to the generated CSS block — negligible impact.
- **NFR-05 Scope:** Feature applies to the full-page bundle widget only. The product-page widget's promo banner (disabled by default) is out of scope.

---

## User Stories (high level)

- As a **merchant**, I want to pick an image from my store's files as the promo banner background so that my bundle landing page looks on-brand.
- As a **merchant**, I want to clear the background image and revert to a solid colour so that I can switch styles quickly.
- As a **merchant**, I want to search my uploaded files by name so that I can find the right image quickly in a large library.

---

## Scope

### In Scope
- File picker UI in DCP → Promo Banner Settings (full-page only).
- Shopify Admin `files` query integration (read-only).
- `promoBannerBgImage` field added to `PromoBannerSettings` interface and `promoBannerSettings` Json column.
- `--bundle-promo-banner-bg-image` CSS variable emitted from `css-variables-generator.ts`.
- Widget CSS update to use `background-image` when variable is set.
- Widget bundle rebuild.

### Out of Scope
- Uploading new images from within the app (merchants upload via Shopify admin).
- Product-page bundle promo banner.
- Video or GIF backgrounds.
- Multiple background images or carousel.
- Overlay tint/opacity controls (future enhancement).

---

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Shopify CDN URL changes after image replacement | Low | URL is re-fetched from picker on each save |
| Large file libraries slow the picker | Medium | Paginate at 25, add filename search filter |
| Image URL becomes inaccessible (file deleted from Shopify) | Low | Fallback to `promoBannerBgColor` solid colour in CSS |
| Widget CSS specificity conflict with theme background-image | Low | Scope selector to `.bundle-widget .promo-banner` |
