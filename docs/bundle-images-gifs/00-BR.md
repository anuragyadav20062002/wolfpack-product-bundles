# Business Requirement: Per-Bundle Images & GIFs

**Feature ID:** bundle-images-gifs
**Created:** 2026-02-20

---

## Executive Summary

Merchants need to set a unique promo banner background image per full-page bundle. The previous global DCP setting has been removed; this feature restores the capability as per-bundle configuration through the existing bundle configure page.

## Problem Statement

**Who:** Merchants managing multiple full-page bundles (e.g., "Summer Kit", "Gift Set", "Starter Bundle")

**Pain point:** The removed DCP promo banner background image was global — one image for all full-page bundles. Merchants with multiple bundles need distinct visual branding per bundle (seasonal themes, product-specific imagery, promotional artwork). There is currently no way to set any background image.

## Research Findings

### Approach A: Extend Bundle Prisma model (new column)
- Add `promoBannerBgImage String?` directly to the `Bundle` model
- Requires a Prisma migration (one `ALTER TABLE` adding a nullable column)
- Clean, typed, queryable, fits existing patterns (Bundle already has nullable `shopifyPageHandle`, `shopifyPageId`, `templateName`)
- **Verdict: ✅ Recommended** — simplest, most straightforward, zero risk of JSON schema drift

### Approach B: Store in BundlePricing.messages JSON
- Extend the existing `messages` JSON blob on `BundlePricing`
- No migration needed
- Creates a type-safety gap; `messages` is meant for discount messaging copy, not media assets
- JSON blobs grow unbounded and become hard to query/index
- **Verdict: ❌ Rejected** — wrong semantic home, poor type safety

### Approach C: Separate BundleMedia model
- New `BundleMedia` table with `bundleId`, `type`, `url` rows
- Extensible for future hero images, step illustrations, GIFs
- Overkill for a single nullable URL; adds join complexity for a query path that runs on every page load
- **Verdict: ❌ Rejected for now** — premature abstraction; can be added later if media requirements grow

## Functional Requirements

- **FR-01:** A new "Images & GIFs" nav item appears in the Bundle Setup card on the full-page bundle configure page (not on product-page bundle configure)
- **FR-02:** The section contains a "Promo Banner Background" field using the existing `FilePicker` component (browse Shopify Content → Files + upload)
- **FR-03:** Selecting or uploading an image persists `promoBannerBgImage` to the `Bundle` DB record on save
- **FR-04:** Removing the image sets `promoBannerBgImage` to `null`
- **FR-05:** The image URL is included in the `bundle_ui_config` metafield after every save so the widget can access it
- **FR-06:** The widget reads `promoBannerBgImage` from the selected bundle config and applies `--bundle-promo-banner-bg-image: url('...')` inline on the promo banner element
- **FR-07:** Save/discard follows the existing App Bridge SaveBar pattern on the configure page

## Non-Functional Requirements

- **NFR-01:** Existing full-page bundles without a `promoBannerBgImage` remain unaffected (the CSS variable falls back to `none`)
- **NFR-02:** Product-page bundle configure page is completely untouched
- **NFR-03:** No changes to the DCP design settings pipeline
- **NFR-04:** `write_files` scope is already present in both `shopify.app.toml` files — no scope changes needed

## User Stories (high level)

- As a merchant, I want to set a unique background image for each full-page bundle's promo banner so that each bundle has distinct visual branding
- As a merchant, I want to upload new images directly from the bundle configure page so that I don't need to switch to Shopify Files manually
- As a merchant, I want to remove the background image so that the banner reverts to a plain background color

## Scope

### In Scope
- "Images & GIFs" section in full-page bundle configure page
- Promo Banner Background sub-section with FilePicker
- Prisma migration for `promoBannerBgImage String?`
- `bundle_ui_config` metafield updated to include the field
- Widget JS reading and applying the CSS variable

### Out of Scope
- Product-page bundle configure page
- Hero images, step illustrations, GIF overlays (placeholder note only)
- Image cropping, resizing, or CDN transformation
- DCP global promo banner image (intentionally removed)

## Risks

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Prisma migration in production causes downtime | Low | Column is nullable; no existing rows affected |
| Widget JS bundle not rebuilt after JS changes | Medium | Enforce `npm run build:widgets` in implementation checklist |
| Metafield payload size increase | Low | Single URL string adds < 100 bytes |
