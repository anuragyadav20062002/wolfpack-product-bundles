# Business Requirement: PDP Bundle Redesign

**Feature ID:** pdp-bundle-redesign
**Status:** BR Complete
**Created:** 2026-03-18

---

## Executive Summary

Redesign the product-page bundle (PDP) widget to match the Skai Lama reference UX exactly, add automated theme template installation, extend free gift and default-product features to PDP, and fully wire the Design Customization Panel (DCP) so every visual element is merchant-configurable.

---

## Problem Statement

Our current PDP widget is functionally incomplete and visually far behind the Skai Lama reference:

| Dimension | Current State | Target (Skai Lama parity) |
|---|---|---|
| Slot cards | Plain "+" icon with "Step N" label, no design | Styled cards with SVG icon, step name in theme color, dashed/solid border |
| Default product (compulsory) | Not visually distinct | Pre-filled slot with blue border, product image, always included |
| Free gift step | **Not supported in PDP at all** | Ribbon icon on slot, promo heading in modal, dark navy tab |
| Modal/bottom-sheet | Opens but lacks Skai Lama visual design | Slides up, rounded top, step tabs, yellow-bordered product cards, floating Prev/Next pill |
| DCP | Settings exist but not all applied to widget HTML | All colors, borders, radii fully driven by CSS variables from DCP |
| Theme template | `ensureProductBundleTemplate()` exists but not auto-called from configure page | Auto-installed when merchant assigns a product to the bundle |
| Product card (filled) | Basic | Product image + name + blue border, remove button |

**Who experiences it:** Merchants using our app expect a polished storefront experience comparable to Skai Lama. The current PDP widget reflects poorly on the product.

## Research Findings

### Reference Design Analysis (Chrome DevTools inspection)

**Skai Lama Toy Bundle** (`ultimate-soft-toy-bundle-new`) — inspected live:

**Product page slot cards:**
- Container: CSS grid, fits into Shopify product info column (~375px wide on mobile)
- Empty slot: `border: 2px solid rgb(146,146,146)` or `2px dashed rgb(30,58,138)`, `borderRadius: 10px`, `width: ~114px`, `height: 200px`
- SVG icon wrapper: circular, 80×80px with 10px padding
- Step/category label: `color: rgb(69,150,227)` (blue), below icon
- Default product (Gift Box): `.gbbMixSelectedProductCard` — `border: 2px solid rgb(69,150,227)`, `borderRadius: 10px`, `padding: 5px`, product image 100×100, name in blue
- Free gift slot: same dashed card + red ribbon SVG overlay in top-right corner, "Free Cap" label
- Add Bundle to Cart: `bg: rgb(69,150,227)`, `color: white`, `borderRadius: 40px`, `padding: 14px 10px`, full-width, disabled when incomplete

**Selection modal (bottom-sheet):**
- Position: `fixed; top: 109px; left: 0; width: 100%; borderRadius: 15px 15px 0 0`
- Background: `rgb(244,249,249)` (very light teal)
- Overlay: `rgba(30,30,30,0.5)`, full screen
- **Header:** step tabs (pill shape, `borderRadius: 40px`, inactive=outlined blue, active=filled blue), discount text `fontSize:16px fontWeight:500 color:#444`
- **Body:** 4-column grid (`gridTemplateColumns: repeat(4,254px)`, `gap:30px`), bottom padding `~127px` to clear footer
- Product card: `bg:white; border:2px solid rgb(255,202,67); borderRadius:16px; padding:12px` — yellow/gold border
- Product title: `color:rgb(69,150,227); fontSize:16px; fontWeight:700`
- "Add to Cart" button: `bg:rgb(69,150,227); borderRadius:40px; padding:6px 10px; color:white; fontSize:16px`
- **Footer (floating pill):** `bg:rgba(30,30,30,0.8); borderRadius:15px; width:300px; height:80px`, Prev/Next white buttons + cart counter white pill

**SWAG Bundle free gift (Free Cap tab):**
- Tab active: `bg:rgb(30,58,138)` (dark navy), `borderRadius:8px` (NOT pill — square-ish)
- Promo heading: "Get a cap worth $15 absolutely free!" — bold, prominent
- Sub-heading: "Add 3 product(s) to get 1 of them at 100% off!"
- Product cards: `border:1px solid rgb(227,227,227)` (light gray), `borderRadius:12px` — NO gold border
- Last step shows "DONE" button instead of "NEXT"

### Codebase Analysis

**What already exists:**
- `BundleWidgetProductPage` class in `app/assets/bundle-widget-product-page.js` (1600+ lines): has `renderUI()`, `renderSteps()`, `createEmptyStateCard()`, `createSelectedProductCard()`, `openModal()`, modal tab/product rendering
- CSS in `extensions/bundle-builder/assets/bundle-widget.css` with `.bw-bs-*` (bottom-sheet) and `.bw-slot-card*` classes
- `ensureProductBundleTemplate()` in `widget-theme-template.server.ts` — creates `templates/product.product-page-bundle.json`
- DCP with 259 settings including `productCardBorderRadius`, `buttonBgColor`, `emptyStateCardBorderColor`, etc.
- `bundle-product-page.liquid` block with CSS variable injection

**What's missing:**
- Free gift step support in PDP widget (only exists in FPB)
- Visual design of slot cards doesn't match reference (no SVG icons, wrong colors/borders)
- Modal design doesn't match reference (wrong card borders, no floating footer pill)
- `ensureProductBundleTemplate()` is never auto-called from the configure route
- Default product (isDefault step) rendering not visually distinct in PDP
- DCP CSS variables exist but not all applied to the rendered HTML elements

## Functional Requirements

**FR-01 — Slot Card Redesign:** Empty state slot cards must display a configurable SVG/icon placeholder, step name in theme primary color, and DCP-controlled border style (dashed or solid, configurable color).

**FR-02 — Filled Product Card:** Selected product slot must show: product image, product title in theme primary color, blue border, a remove (×) button.

**FR-03 — Default Product Card:** Steps marked `isDefault=true` must render pre-filled with the default product image + name + themed border. Cannot be removed by the merchant's customer. Not clickable to open modal.

**FR-04 — Free Gift Slot:** Steps marked `isFreeGift=true` must:
- Show a ribbon/gift SVG overlay on the slot card corner
- Open a modal tab with a promo heading ("Get [name] absolutely free!")
- Use dark navy tab styling (distinct from regular steps)
- Show product cards with gray border (not gold) in the modal
- Show "Done" instead of "Next" on the last step

**FR-05 — Bottom-Sheet Modal Redesign:** The selection modal must:
- Slide up from bottom with rounded top corners
- Show step tabs as pill buttons (active=filled, inactive=outlined)
- Show product grid with yellow/gold card borders (except free gift: gray)
- Show floating Prev/Next navigation pill at bottom
- Auto-advance to next incomplete step after selection
- Show discount progress text ("Add N more products to get bundle at $X")

**FR-06 — Add Bundle to Cart Button:** Full-width pill button, disabled (grayed) until all required steps are complete, uses DCP primary button color.

**FR-07 — Automated Theme Template:** When a merchant configures a PDP bundle and clicks a "Add to Storefront" CTA (or on first save), call `ensureProductBundleTemplate()` to write `templates/product.product-page-bundle.json` to the active theme. This is idempotent — safe to call multiple times.

**FR-08 — DCP Full Coverage:** Every color, border, radius, font size visible in the widget must have a corresponding CSS variable served from `/api/design-settings`. Widget JS must read and apply these variables.

**FR-09 — Free Gift in PDP (Parity with FPB):** PDP bundles must support `isFreeGift` and `isDefault` step types with the same semantics as FPB bundles.

---

## Non-Functional Requirements

**NFR-01 — No Regression:** Existing PDP bundles (currently configured) must continue to work. The widget must be fully backward-compatible.

**NFR-02 — DCP CSS Variables:** All design tokens delivered as CSS custom properties via the DCP endpoint. No hardcoded colors in widget JS.

**NFR-03 — Widget Version Increment:** After any JS/CSS change, `WIDGET_VERSION` must be bumped and bundle rebuilt.

**NFR-04 — Responsive:** Widget must work on mobile (< 480px) — slot cards stack vertically, modal fills screen width with 2 columns on mobile.

**NFR-05 — Performance:** Template installation is async/idempotent; no blocking of configure page save operation.

## User Stories (High Level)

- As a **merchant**, I want my product bundle widget to look polished and on-brand so that my customers are more likely to complete their bundle purchase.
- As a **merchant**, I want the bundle widget installed automatically on my product page so I don't need to navigate the Theme Editor.
- As a **merchant**, I want to offer a free gift as part of my bundle (e.g., a free cap when buying 3 items) so I can drive higher AOV.
- As a **merchant**, I want to include a compulsory product in my bundle (e.g., a gift box always included) so every bundle has a consistent base.
- As a **customer**, I want to clearly see which steps are empty vs filled so I know what I still need to choose.
- As a **customer**, I want the product selection experience to feel smooth and visual so I enjoy building my bundle.

## Scope

### In Scope
- Visual redesign of PDP widget slot cards (empty, filled, default, free gift)
- Bottom-sheet modal visual redesign (tabs, product cards, footer pill)
- Free gift step support in PDP widget (display + behavior)
- Default/compulsory product step in PDP (pre-filled, non-removable)
- Automated `product.product-page-bundle.json` template installation from configure page
- DCP full coverage for PDP widget
- Widget version bump + bundle rebuild

### Out of Scope
- Changes to the FPB (full-page bundle) widget
- New DCP settings (only wire existing 259 settings to widget)
- Cart transform changes (free gift pricing already handled)
- Admin configure route redesign
- Shopify deploy (merchant must run manually)

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| Template write fails if merchant's theme blocks the REST API | Low | Graceful fallback: show Theme Editor link (same pattern as FPB) |
| Free gift step PDP causes cart transform regression | Medium | Cart transform already handles `isFreeGift`; ensure PDP sends same `_bundle_step_type` property |
| DCP CSS variables conflict with theme CSS | Low | All variables scoped under `.bundle-widget-container` |
| Widget redesign breaks existing bundle configurations | Low | Empty-state and filled-state rendering is purely visual; no data model changes |
| Mobile layout broken | Medium | Test at 375px, 480px, 768px, 1280px breakpoints |
