# Business Requirements — Beco-Style Pricing Tier Selection for Full-Page Bundle Widget

**Document Type:** Business Requirement (BR)
**Feature ID:** fpb-tier-selection
**Status:** Draft
**Created:** 2026-03-17
**Author:** Feature Pipeline

---

## 1. Executive Summary

Merchants selling bundles with tiered quantity-based pricing (e.g. "Buy 2 for ₹499", "Buy 3 for ₹699") currently have no way to surface those tiers in the full-page bundle widget. Each tier is a separate Bundle record in the database, but the storefront widget only shows one bundle at a time — determined by a single `bundle_id` setting or page metafield.

This feature adds a **tier pill selector** at the top of the full-page widget. The merchant configures up to 4 tiers (each mapping a human-readable label to a bundle ID) in the Shopify Theme Editor. The widget renders pill buttons and re-initialises itself with the appropriate bundle when the shopper clicks a different tier.

The reference implementation is the Beco brand's bundle page, which shows three pills: "Buy 2 @499 ›", "Buy 3 @699 ›", "Buy 4 @849 ›". This feature brings parity with that design pattern.

---

## 2. Problem Statement

### Current State
- The full-page bundle widget accepts exactly one `bundle_id`.
- Merchants who want to offer quantity-based pricing must create separate landing pages per tier, fragmenting the shopper journey and requiring separate marketing links for each tier.
- There is no in-widget mechanism for a shopper to compare tiers and self-select.

### Desired State
- A single bundle page can surface 2–4 pricing tiers as pill buttons.
- Clicking a tier pill replaces the widget's product selection experience with the products and pricing for that tier's bundle.
- The page URL, promo banner, and all other page-level elements remain the same — only the widget content changes.
- Merchants configure everything in the Shopify Theme Editor; no custom coding is required.

## 5. Scope

### In Scope
- Tier pill bar rendered above the widget's promo banner (or at the very top of the widget container) on the full-page bundle widget.
- Up to 4 configurable tiers, each with a label and a bundle ID.
- Widget re-initialises the product selection area when a pill is clicked.
- Loading state during tier switch (spinner, pill disabled temporarily).
- CSS variables compatible with the existing Design Control Panel (DCP) system.
- Responsive behaviour (pills wrap or scroll horizontally on mobile).
- Pills are hidden when fewer than 2 tiers are configured.
- No new Prisma models, no new API routes, no DB migrations.

### Out of Scope
- Automatic tier generation from a single bundle (requires schema redesign — deferred).
- DCP Design Control Panel settings for pill colours (colours are configurable via existing CSS variable overrides; DCP integration is a future enhancement).
- URL-hash–based deep-linking to a specific tier on page load (deferred).
- A/B testing of tier layouts.
- Order status / thank-you page tier display.

## 6. User Stories

### Merchant — Configuration
**US-01:** As a merchant, I want to enter up to 4 tier labels and bundle IDs in the Theme Editor so that I can offer tiered pricing on a single bundle page without writing code.

**US-02:** As a merchant, I want the tier pills to be hidden automatically if I configure only one tier, so I do not need to manually toggle a display setting.

### Shopper — Selection
**US-03:** As a shopper, I want to see pill buttons showing pricing tiers at the top of the bundle page so that I can quickly understand my quantity/price options.

**US-04:** As a shopper, I want the active tier pill to be visually distinct (highlighted) so that I always know which tier I am currently viewing.

**US-05:** As a shopper, I want clicking a tier pill to immediately update the bundle contents (products, pricing, steps) so that I can compare tiers without navigating away.

**US-06:** As a shopper, I want a brief loading indicator while the new tier's data loads so that I know the page is responding to my action.

**US-07:** As a shopper on mobile, I want the tier pills to be easily tappable and not overflow the screen.

## 7. Constraints and Assumptions

### Constraints
- Implementation must use existing Bundle records (Option A: multiple Bundle records per tier). No DB schema changes.
- Tier configuration lives entirely in the Liquid block settings (Theme Editor). No app-side admin UI changes.
- The widget must remain backward-compatible: pages with a single `bundle_id` and no tier settings must continue to work exactly as today.
- The feature must not break the existing `data-bundle-id` resolution priority chain (custom metafield → page handle → app metafield → block setting).

### Assumptions
- Merchants will create and save each tier's bundle in the Wolfpack admin dashboard before configuring the tier pills.
- Bundle IDs are stable strings (cuid format) that do not change after creation.
- The Shopify app proxy endpoint `/apps/product-bundles/api/bundle/{id}.json` is already capable of serving each tier's bundle data independently.
- A tier switch is a full re-initialisation of the widget (clear selections, reload bundle data, re-render UI) — partial state preservation is not required in v1.

---

## 8. Reference Design

Beco's bundle page demonstrates the target UX:
- Three pill buttons at the top of the widget area: "Buy 2 @499 ›", "Buy 3 @699 ›", "Buy 4 @849 ›".
- Active pill: bright green background (#00FF00), 1px solid black border, 8px border-radius, approximately 60px height, 355px width.
- Inactive pill: light green background (rgb(242, 250, 238)), same border and radius.
- Clicking a pill loads a different bundle (different product catalog, different pricing) into the same widget container.
- The promo banner image changes per tier (nice-to-have for v1; out-of-scope for initial implementation).

## 10. Risks

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Merchant enters invalid bundle ID for a tier | Medium | Medium — widget shows error for that tier | Validate bundle ID presence; show error state on pill without crashing the whole widget |
| Slow API response on tier switch | Low | Medium — shopper sees delayed content | Show loading spinner; disable pills during fetch |
| CSS conflict with theme styles on pill bar | Low | Low — visual glitch only | Use scoped class names prefixed with `bundle-tier-` |
| Widget version cache: old bundled JS served after deploy | Low | Medium — stale code | Follow mandatory version increment + deploy procedure per CLAUDE.md |
