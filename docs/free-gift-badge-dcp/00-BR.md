# Business Requirement: Free Gift Badge — DCP Asset Picker

## Executive Summary
Merchants need a way to upload or select a custom image (PNG or SVG) from their Shopify store's own asset library as the free-gift badge that appears on locked gift-step slots in the bundle widget — replacing the current plain text field that requires a manual CDN URL.

## Problem Statement
Free gift steps in bundle widgets display a badge/ribbon on slot cards to indicate a "gift" item. The current DCP only exposes a raw text input for `freeGiftBadgeUrl`, expecting merchants to paste in a CDN URL manually. This is friction-heavy, error-prone, and inconsistent with how promo banner images are handled (which already use a polished Shopify-integrated FilePicker). Additionally:
- The badge picker is gated behind the "bottom-sheet" widget style toggle, making it unreachable to merchants using the classic layout even though free gift slots exist in both modes.
- The "Widget Style" DCP subsection has no navigation sidebar entry, so the feature is effectively undiscoverable.
- The full-page bundle widget has no image-badge support at all — only a hardcoded "Free" text label.

**Who is affected:** Any merchant who has configured a bundle step as a free gift (`isFreeGift: true`), across both product-page and full-page bundle types.

## Research Findings

### Approach A: Extend existing FilePicker component (reference implementation)
Reuse `app/components/design-control-panel/settings/FilePicker.tsx`, which already integrates with `app.store-files` (browse) and `app.upload-store-file` (staged upload + poll) routes. The component handles the full file lifecycle: browse, upload, preview thumbnail, adjust, remove. This is identical to how promo banner background images work.

**Pros:** Zero new infrastructure; fully consistent UX; already supports SVG + PNG via the `query: "media_type:IMAGE"` filter on the Shopify Files API.
**Cons:** None — it's a direct re-use.

### Approach B: New file upload widget using Shopify App Bridge FilePicker
Use the App Bridge `ResourcePicker` or a custom `<input type="file">` flow.

**Pros:** Slightly more native feel.
**Cons:** Significant new work; inconsistent with the existing promo-banner pattern already in the codebase. Rejected.

**Recommended:** Approach A — reuse `FilePicker.tsx` exactly as done for the promo banner.

## Functional Requirements (FRs)

- **FR-01:** The DCP "Widget Style" subsection must be accessible from the navigation sidebar for both bundle types.
- **FR-02:** The free gift badge image picker must be visible for all widget style modes (classic + bottom-sheet), not only bottom-sheet.
- **FR-03:** The image picker must allow the merchant to browse existing PNG/SVG files from their Shopify store's file library.
- **FR-04:** The merchant must be able to upload a new PNG or SVG directly from the picker.
- **FR-05:** When a badge image is selected or uploaded, a thumbnail preview must be shown inline in the DCP.
- **FR-06:** The merchant must be able to remove the badge image (revert to the built-in SVG ribbon / "Free" text).
- **FR-07:** The selected badge image URL must be saved via the DCP save flow and persisted to the `DesignSettings.freeGiftBadgeUrl` column.
- **FR-08:** The CSS variable `--bundle-free-gift-badge-url` must be emitted with the correct `url("...")` value when a badge is set, or `none` when cleared.
- **FR-09:** The product-page widget (`bundle-widget-product-page.js`) must render the `<img>` badge when the CSS variable is set (already implemented — verify only).
- **FR-10:** The full-page widget (`bundle-widget-full-page.js`) must read `--bundle-free-gift-badge-url` and render an `<img>` alongside or replacing the "Free" text badge when set.
- **FR-11:** The feature must apply to both `product_page` and `full_page` DCP instances.

## Non-Functional Requirements (NFRs)

- **NFR-01:** No new Prisma columns or DB migration required — `freeGiftBadgeUrl` column already exists.
- **NFR-02:** Widget JS changes require `WIDGET_VERSION` bump and `npm run build:widgets`.
- **NFR-03:** Shopify CSS file size limit (100,000 B) must not be exceeded after any CSS additions.
- **NFR-04:** The FilePicker must not block the DCP save bar — the URL value is managed through the same `onUpdate("freeGiftBadgeUrl", url)` mechanism as all other DCP settings.

## User Stories (high level)

- As a merchant, I want to select a PNG or SVG from my Shopify store files as the free-gift badge so that my bundle looks on-brand.
- As a merchant, I want to upload a new badge image directly from the DCP so I don't have to leave Shopify admin.
- As a merchant, I want to remove the badge image and revert to the default ribbon/text so I can reset my customization.
- As a merchant, I want the badge to apply in both my classic and bottom-sheet bundle widget layouts so I don't have to switch styles to access this feature.

## Scope

### In Scope
- Replace `<TextField>` with `<FilePicker>` in `WidgetStyleSettings.tsx`
- Remove the `{isBottomSheet && ...}` gate around the badge field
- Add "Widget Style" navigation entry to `NavigationSidebar.tsx` for both bundle types
- Full-page widget: add `--bundle-free-gift-badge-url` image rendering on `.fpb-free-badge`
- WIDGET_VERSION bump + rebuild

### Out of Scope
- Per-step badge customization (one badge per shop, not per step)
- Crop/adjust UI for the badge image (badge is small — crop not needed)
- Animated GIF badges (PNG/SVG only; CSS `url()` already handles static images)
- Promo banner image functionality changes

## Risks
| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Full-page widget CSS file exceeds 100KB after badge CSS | Low | Run `wc -c` after changes; strip comments if needed |
| Store files API returns no SVG files (filtered out) | Medium | Test query includes `media_type:IMAGE` — SVGs are included in Shopify's IMAGE media type |
| Widget rebuild breaks other toast/DCP changes | Low | Run full `npm run build:widgets` + tsc before commit |
