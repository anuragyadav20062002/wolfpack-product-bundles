# Business Requirement: Default Lottie Loading Animation

## Executive Summary
Replace the plain CSS spinner fallback with a polished Lottie JSON animation that plays by default when a merchant has not uploaded a custom loading GIF for their bundle.

## Problem Statement
When a merchant creates a bundle and does not configure a custom loading GIF, the loading overlay displays a minimal white CSS spinner (rotating ring). This looks generic and unpolished. A branded, smooth Lottie animation would provide a significantly better user experience out-of-the-box, without requiring merchants to source and upload their own animation.

## Research Findings

### Approach A: CDN-loaded `lottie-player` web component + embedded JSON
- Embed Lottie JSON data (~2-5KB) directly in the widget JS as a constant
- Lazy-load the `@lottiefiles/lottie-player` web component (~70KB gzipped) from CDN when needed
- Show CSS spinner as immediate fallback, swap to Lottie once player loads
- **Pros:** True Lottie playback, minimal bundle size impact, progressive enhancement
- **Cons:** External CDN dependency, potential CSP issues on some stores, brief flash before Lottie renders

### Approach B: Bundle `lottie-web` light build into widget IIFE
- Include `lottie-web/build/player/lottie_light.min.js` (~40KB gzipped) directly in the widget bundle
- Embed Lottie JSON data as a constant
- Render via `lottie.loadAnimation()` API directly
- **Pros:** Fully self-contained, no CDN dependency, no CSP issues, instant Lottie rendering
- **Cons:** Adds ~150KB unminified / ~40KB gzipped to each widget bundle; increased parse time on mobile

### Approach C: Convert Lottie to inline SVG with SMIL animations
- Pre-convert the chosen Lottie animation to a standalone animated SVG
- Embed the SVG markup directly in the widget rendering code
- **Pros:** Zero JS overhead, instant rendering, completely self-contained
- **Cons:** Cannot dynamically play Lottie JSON — it's a static conversion; limited to simple animations; defeats the purpose of "using Lottie JSON"

### **Recommended: Approach B — Bundle lottie-web light into widget**
- **Rationale:** Shopify theme extensions are served via Shopify's CDN with gzip/brotli compression. The ~40KB gzipped increase is acceptable for a polished loading experience. This avoids external CDN dependencies, CSP issues, and flash-of-unstyled-content. The widget bundles are already built via an IIFE bundler (`npm run build:widgets`), making this integration straightforward. The lottie-web light build (SVG renderer only) is purpose-built for this use case.

## Functional Requirements (FRs)
- **FR-01:** When a bundle has no custom `loadingGif` set, the loading overlay displays a default Lottie animation instead of the CSS spinner
- **FR-02:** When a merchant uploads a custom GIF via the FilePicker, it takes precedence over the default Lottie animation (existing behavior preserved)
- **FR-03:** The default Lottie animation must work in both product-page and full-page widget types
- **FR-04:** The Lottie animation must loop smoothly and render at the same max dimensions as custom GIFs (120x120px)
- **FR-05:** The existing overlay behavior (fade in/out, semi-transparent background, z-index) remains unchanged
- **FR-06:** The default animation is embedded in the widget code — no additional network requests required

## Non-Functional Requirements (NFRs)
- **NFR-01:** Performance — Widget bundle size increase must not exceed ~50KB gzipped per bundle
- **NFR-02:** Compatibility — Must work in all browsers supported by Shopify storefronts (Chrome, Firefox, Safari, Edge — last 2 versions)
- **NFR-03:** Backward compatibility — Existing bundles with custom GIFs must continue to work identically
- **NFR-04:** The Lottie JSON for the default animation should be small (<5KB)

## User Stories (high level)
- As a **merchant**, I want my bundles to show a polished loading animation by default so that my customers have a professional experience without me needing to configure anything
- As a **merchant**, I want to be able to replace the default animation with my own GIF so that I can match my store's branding
- As a **customer**, I want to see a smooth, visually appealing loading indicator when interacting with a bundle so that I know the page is working

## Scope

### In Scope
- Embedding a default Lottie JSON animation in both widget bundles
- Adding lottie-web light as a bundled dependency in the widget build process
- Modifying `showLoadingOverlay()` to render Lottie when no custom GIF URL is provided
- Selecting/creating an appropriate default loading animation

### Out of Scope
- Allowing merchants to upload custom Lottie JSON files (they can still upload GIFs)
- Admin UI changes (no new settings needed — existing FilePicker override is sufficient)
- Database schema changes
- Metafield changes

## Risks
| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Bundle size increase impacts page load | Low | lottie-web light is ~40KB gzipped; Shopify CDN compression minimizes impact |
| Lottie animation doesn't render in older browsers | Low | lottie-web has excellent browser support; CSS spinner remains as ultimate fallback |
| Build process complexity increases | Low | lottie-web is a well-maintained npm package; build script already uses esbuild |
